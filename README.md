# RunFaster5K-SQL
 Sistema de Gestão de Corridas de Rua - "RunFaster 5K"1. Contexto e ObjetivosEste projeto implementa um Banco de Dados Relacional (SGBD MySQL) para gerenciar eventos de corrida de rua, participantes e seus resultados. O foco principal é a aplicação prática da Linguagem de Definição de Dados (DDL) e da Linguagem de Manipulação de Dados (DML), garantindo a integridade referencial e a normalização do modelo lógico.RequisitoTaxonomia de Bloom/FinkStatusModelagem LógicaIntegrar normalização e integridade.✅Criação de ScriptsCriar scripts SQL completos e bem estruturados.✅Manipulação de DadosExecutar comandos DML (INSERT, SELECT, UPDATE, DELETE).✅EntregaRepositório público no GitHub com README completo.✅2. Análise dos Scripts DMLO arquivo dml_scripts.sql contém a implementação da DML, demonstrando a manipulação de dados e as regras de integridade.4.1. Consultas Complexas (SELECT)Foram implementadas as seguintes consultas, utilizando WHERE, ORDER BY, LIMIT e JOINs para ligar as 4 tabelas:Ranking por Categoria na Edição Verão: Une RESULTADO e CATEGORIA para classificar os corredores dentro de sua faixa etária no evento id=1.Melhores Tempos (Top 5): Consulta que une CORREDOR, RESULTADO e CORRIDA, ordenando pelo tempo oficial de forma ascendente (ASC) e limitando os resultados com LIMIT 5.Corredores por Categoria: Consulta que busca o nome do corredor e o nome do evento para todos aqueles que pertencem a uma categoria específica ('25-29 anos'), utilizando uma subconsulta.4.2. Manipulação de Dados (UPDATE e DELETE)O script utiliza chaves primárias ou chaves únicas na cláusula WHERE em todos os comandos UPDATE e DELETE para respeitar o Safe Update Mode (Erro 1175) e garantir que apenas os registros pretendidos sejam modificados.UPDATEs: Atualização de posição de resultado, alteração de dados de contato do corredor (usando WHERE cpf), e correção da localização de uma corrida (usando WHERE id_corrida).DELETEs: Demonstra a exclusão de dados, incluindo a necessidade de deletar os registros filhos (RESULTADO) antes de deletar o registro pai (CORREDOR) para manter a integridade referencial.5.
